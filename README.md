# Лабораторная работа №12. Жизненный цикл Activity (Activity Lifecycle)

## Описание приложения
Мы создадим приложение **"Grade Clicker"** — интерактивный кликер для набора
баллов по курсу "Разработка мобильных приложений".

**Механика:**
- Пользователь кликает по изображению, набирая баллы
- За каждый клик начисляется **+5 баллов**
- При достижении определённого количества баллов изменяется: Отображаемая оценка; Фоновое изображение; Цвет интерфейса

**Система оценок:**
-   **0-49 (Неудовлетворительно):** "Продолжайте работать!"
-   **50-69 (Удовлетворительно):** "Вы получили зачёт!"
-   **70-89 (Хорошо):** "Отличная работа!"
-   **90-100 (Отлично):** "Вы успешно сдали курс!"

## Цель работы
Изучить жизненный цикл Activity в Android и научиться правильно
обрабатывать переходы между состояниями. В результате лабораторной работы будет
создано интерактивное приложение "Grade Clicker" (Кликер оценок), которое
демонстрирует работу lifecycle callbacks и сохранение состояния при изменении
конфигурации устройства.

## Контрольные вопросы

**1. Какие три метода вызываются при запуске приложения?**

При запуске приложения последовательно вызываются три метода: `onCreate()`, `onStart()` и `onResume()`. `onCreate()` создаёт Activity, `onStart()` делает её видимой для пользователя, а `onResume()` предоставляет ей фокус ввода, делая активной.

**2. Чем отличается onPause() от onStop()?**

`onPause()` вызывается, когда Activity теряет фокус, но остаётся частично видимой (например, при появлении диалогового окна или открытии шторки уведомлений). `onStop()` же вызывается, когда Activity становится полностью невидимой для пользователя (например, при сворачивании приложения или открытии нового Activity поверх текущего).

**3. Когда вызывается onRestart()?**

Метод `onRestart()` вызывается перед `onStart()`, когда Activity перезапускается после того, как была остановлена (находилась в состоянии `Stopped`). Это происходит, когда пользователь возвращается к Activity, которое было в фоне, но не уничтожено системой.

**4. Что происходит при повороте экрана?**

При повороте экрана происходит изменение конфигурации устройства (configuration change). По умолчанию Android полностью уничтожает текущую Activity (вызываются `onPause()`, `onStop()`, `onDestroy()`) и затем создаёт её заново с нуля (вызываются `onCreate()`, `onStart()`, `onResume()`). Это приводит к потере всех данных, хранящихся в обычных переменных.

**5. В чём разница между remember и rememberSaveable?**

`remember` сохраняет значение только при рекомпозиции (перерисовке интерфейса в рамках одного экземпляра Activity). `rememberSaveable` же идёт дальше: он сохраняет значение как при рекомпозиции, так и при изменении конфигурации (например, повороте экрана), когда Activity пересоздаётся.

**6. Почему onCreate() должен вызывать super.onCreate()?**

`super.onCreate(savedInstanceState)` необходим для вызова родительской реализации метода в классе `AppCompatActivity` (или `Activity`). Этот вызов выполняет важную внутреннюю работу по инициализации Activity, восстановлению её состояния (из `savedInstanceState`) и настройке окружения. Без этого вызова Activity не сможет корректно функционировать, и приложение, скорее всего, упадёт с ошибкой.

**7. Для чего используется класс Log?**

Класс `Log` используется для вывода отладочных сообщений в панель Logcat в Android Studio. Это позволяет разработчику отслеживать выполнение программы, проверять значения переменных и, как в этой лабораторной работе, видеть порядок вызова методов жизненного цикла Activity. Он поддерживает разные уровни важности: `Log.v()`, `Log.d()`, `Log.i()`, `Log.w()`, `Log.e()`.

**8. Какой метод вызывается только один раз за время жизни Activity?**

Метод `onCreate()` вызывается только один раз за весь жизненный цикл Activity — в момент её создания. При последующих перезапусках Activity (например, после возврата из состояния `Stopped`) этот метод уже не вызывается.
